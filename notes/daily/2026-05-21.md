## Core Takeaways

- [ ] **RAG is an evidence chain, not a vector DB product.** Chunking → vectorizing → retrieval → rerank → citation. Break one link, and hallucination just moved from inside the model to inside your retrieval system.
- [ ] **Agent is a constrained execution loop, not autonomy itself.** Goals + tools + state + permissions + stop conditions — all five are required. A model with just tools attached is a talking script executor.
- [ ] **RAG ↔ Agent relationship**: RAG is the Agent's "eyes" (fetching current info before decisions); Agent is RAG's "hands" (executing after research). In a hackathon project, you'll use both together.
