## Core Takeaways

- [ ] **Wallet is not a login button — it's the last confirmation interface before user intent enters on-chain execution.** Three actions, three risk levels: connect wallet (read address) < sign message (prove identity) < send transaction (change chain state).
- [ ] **Smart contracts turn rules into public infrastructure, and bugs into public risk.** Code is visible and callable by anyone. You're not writing normal functions — you're writing programs that manage real assets.
- [ ] **ABI tells you WHAT you can call, not whether calling it is SAFE.** `transfer(address to, uint256 amount)` gives parameter types. It won't tell you if `to` is malicious, if you have enough balance, or if the function is paused.
