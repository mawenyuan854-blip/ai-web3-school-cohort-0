## Core Takeaways

- [ ] **链上系统默认暴露在公开对抗环境里，任何可调用路径都要按攻击面看待。** 普通后端有权限、网络隔离、回滚兜底。合约：代码公开、状态公开、资金公开、攻击者可反复模拟抢跑。
- [ ] **权限必须最小化。** owner、admin、upgrade、pause、withdraw 都要有清楚边界。不要只问"有没有 onlyOwner"——要问 owner 是 EOA 还是多签？有没有 timelock？角色能否相互授予？私钥泄漏最坏结果是什么？
- [ ] **执行前模拟，上线后监控，中间审计。** 安全不是部署那一刻结束，是一整套工程流程。

## Notes & Takeaways

### 为什么学这个

- 链上系统错误成本远高于普通应用：合约部署后资产已在协议里，攻击者直接调公开接口，结果不可回滚
- 安全不是 auditor 一个人的事——产品、前端、后端、合约、运营、AI Agent 都影响安全边界
- 一个错误授权按钮、一个无限 approve、一个不受限的 Agent tool、一条没模拟的交易 → 都是事故入口
- **Web3 安全的核心不是"没有 bug"，而是把可预见风险挡在执行前，执行后快速发现和止损**

### 第一性原理

- 普通后端 vs 合约：
  - 普通后端：权限隔离、网络隔离、人工回滚、数据库修复
  - 合约：代码公开、状态公开、资金公开、攻击者可反复模拟抢跑
- 三条铁律：
  1. **权限最小化**：owner/admin/upgrade/pause/withdraw 边界清楚
  2. **执行前模拟**：交易能不能成功、改什么资产、调哪些合约，提前看见
  3. **上线后监控**：异常转账、参数变更、失败交易、价格波动持续观察

### Reentrancy（重入攻击）

- 合约在外部调用完成前被再次调用，状态被重复利用
- 经典模式：**先转账/外部调用，后更新内部余额** → 恶意合约在回调中再次进入，余额归零前重复提款
- 防护：
  - **Checks-Effects-Interactions**：先检查 → 再更新状态 → 最后外部调用
  - 高风险函数用 reentrancy guard
  - 不在状态未更新前调用不可信合约
  - 测试多合约交互，不只测单个函数

### Access Control（权限控制）

- 决定谁能执行敏感动作（mint、burn、pause、upgrade、withdraw、setOracle、setFee、setRouter、changeOwner）
- 任何一个权限过宽 → 协议规则可能被管理员或攻击者改写
- 检查权限时不止问"有没有 onlyOwner"，要问：
  - owner 是 EOA、多签还是治理合约？
  - 是否有 timelock？
  - 角色能否相互授予？
  - 权限变更是否发出 event？
  - 紧急暂停和恢复由谁控制？
  - 私钥泄漏时最坏结果是什么？

### Audit（审计）

- 审计是外部安全审查，不是安全保证书
- 审计可发现问题，但不能保证协议永远安全
- 审计范围、代码版本、依赖版本、部署参数、升级权限、上线后改动 → 都影响结论
- 读 audit report 至少看：
  - 覆盖了哪些 commit 和合约
  - 哪些问题已修复，哪些被项目方接受风险
  - 是否覆盖经济机制和外部依赖
  - 是否包含测试建议和部署注意事项
  - 上线代码是否和审计代码一致

### Simulation（交易模拟）

- 交易发出前的预演：调用哪个合约、转出哪些 token、会得到什么、gas 多少、是否 revert、是否触发授权变化
- 不能替代安全审计（真实链状态可能在交易打包前变化），但能挡住明显错误：
  - 链 ID 错
  - 合约地址错
  - 授权额度异常
  - 滑点过大
  - 余额不足
  - 调用方法不符合预期

### Monitoring（监控）

- 上线后的安全感知层，发现异常触发响应
- 链上监控关注点：
  - 大额转账或提款
  - 管理员权限变更
  - 合约升级
  - 预言机价格异常
  - 大量失败交易
  - TVL 快速流出
  - 未预期的 event
  - Agent 连续触发高风险工具
- 真正有效的是 **监控 + 响应**：谁收告警、谁能 pause、谁确认误报、恢复流程是什么

### AI × Web3 安全

- AI 让安全边界更复杂：Agent 可解释合约、生成交易、调用工具、执行策略，但也可能读错上下文、被 prompt injection 影响、调用错误工具或生成危险交易
- AI x Web3 安全设计原则：
  - 模型输出和链上执行分开
  - 模型可以建议，工具返回事实
  - policy 限制权限
  - simulation 预演结果
  - human check 确认高风险动作
  - monitoring 记录执行后果
